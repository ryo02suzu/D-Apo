# Handoff: Dentia — 歯科医院 架電CRM（モバイルアプリ）

## Overview
歯科医院向けの「架電（テレアポ）管理CRM」モバイルアプリのUIプロトタイプです。営業担当が医院リストに電話をかけ、結果（ステータス）とメモを記録し、チームでリアルタイムに進捗を共有することを目的としています。iOS想定のスマホ1画面（390×844pt）で、ホーム / 一覧 / 詳細 / 結果入力 / 履歴 / ダッシュボードの6画面とボトムタブで構成されています。

## このデザインファイルについて（重要）
このバンドルに含まれる HTML / JSX ファイルは、**HTMLで作られたデザインリファレンス（プロトタイプ）**です。見た目と挙動の「正解」を示すものであって、本番コードとしてそのまま使うものではありません。

実装タスクは、**これらのHTMLデザインを、ターゲットとなるコードベースの既存環境（React Native / Swift / Flutter / 既存のReact Webなど）で、その環境の確立されたパターン・ライブラリを使って再現すること**です。まだ環境が無い場合は、プロジェクトに最適なフレームワークを選定して実装してください。React + Babelをブラウザで読み込むこの構成は、あくまで素早くプロトタイプを見せるためのものなので、そのまま流用しないでください。

## Fidelity
**High-fidelity（ハイファイ）**です。配色・タイポグラフィ・余白・角丸・影・状態（hover/active/disabled）まで作り込んでいます。ピクセル単位で忠実に再現してください。数値・配色は下記「Design Tokens」と各CSSが正です。

## Screens / Views

ボトムタブは5項目：**ホーム / 一覧 / 次の医院（中央の大きい電話FAB） / 履歴 / ダッシュ**。
中央タブは円形のアクセントカラーFAB（54×54px、中の電話アイコンは白）で、押すと「次に架電すべき医院」の詳細へ遷移します。

### 1. ホーム（HomeScreen）
- **目的**: その日の進捗を一目で把握し、すぐに次の架電を始める。
- **レイアウト**: 縦スクロール。上から ①同僚プレゼンス（任意） ②今日の進捗カード ③次に電話する医院 ④次回予定リスト。区切りは1pxの `--line` divider。
- **コンポーネント**:
  - *プレゼンスピル*: 角丸999px、`--card`背景、左に脈動する小ドット（`--teal`、`pulse` 1.6s）。同僚の現在の操作（「架電中」等）を表示。最大2件。
  - *進捗カード*: 「今日の進捗」ラベル + 「詳細を見る」リンク（`--teal`）。大数字（`<b>` 34px/900）で完了件数、`/ N件`（`--muted`）。下に8px高のプログレスバー（トラック`--line2`、塗り`--teal`、`width`を0.5sで遷移）。さらに3分割スタッツ（ヒアリング / アポ獲得 / ヒアリング率）。セル区切りは縦1px線。
  - *次に電話する医院*: ステータスBadge + 経過時間。医院名（20px/800）、住所（13px/`--muted`）、電話（`tel:`リンク、アイコン+番号）、前回メモ。下に大きな**電話するボタン**（`.btn-primary`：`--teal`背景、白文字18px、角丸14px、padding17px、影あり）。タップで `tel:` 発信し、400ms後に結果入力画面へ遷移。
  - *次回予定*: 時刻（48px幅/800）+ 医院名 + ステータスBadge の行リスト。行区切り1px。

### 2. 一覧（ListScreen）
- **目的**: 医院を検索・絞り込みして架電対象を選ぶ。
- **レイアウト**: 検索バー → エリアチップ行（横スクロール）→ ステータスチップ行 → 件数 → カードリスト（gap 12px）。
- **コンポーネント**:
  - *検索バー*: 角丸12px、1.5px境界`--line2`、虫眼鏡アイコン+input。入力時のみ右にクリアボタン。
  - *チップ*: 角丸999px、未選択は白+`--line2`境界、選択時はステータス色で塗り＋白文字。横スクロール（スクロールバー非表示）。
  - *医院カード（.listcard）*: 角丸14px、1px境界。上段=Badge+「N分前更新/未架電」、医院名（18px/800）、住所、電話リンク。フッターは上1px罫線で区切り、左に「次回予定」「担当者」メタ（アイコン+テキスト）、右に丸い**ミニ電話FAB**（42×42px、`--teal`、白アイコン）。リアルタイム更新で対象カードが `.flash`（`--teal`の3pxリング+`flashPop`アニメ1.4s）。

### 3. 詳細（DetailScreen）
- **目的**: 1医院の架電履歴・基本情報を確認し、結果を追加する。
- **レイアウト**: 戻るヘッダ（右に「編集」リンク）→ 医院名+丸い電話FAB（50×50px）→ 電話リンク → Badge群 → タブ（履歴 / 基本情報）→ 内容 → 下部「架電結果を追加」アウトラインボタン。
- **履歴タブ**: 縦タイムライン。左レール（13px のステータス色リング + 縦stem 2px `--line2`）、右に日時/担当者/Badge とメモ。
- **基本情報タブ**: ラベル78px幅のキー/バリュー行（電話/住所/診療時間/担当者/エリア）、最下部に破線枠の「地図で開く」。

### 4. 結果入力（ResultScreen）
- **目的**: 架電結果のステータスとメモを記録する。
- **レイアウト**: クローズヘッダ → 医院名（中央22px/800）+電話 → 「結果（ステータス）」2列グリッドの選択ボタン → メモtextarea（200字上限・カウンタ）→ 音声入力アウトラインボタン → 下部「保存して次へ」プライマリボタン。
- **状態ボタン（.stbtn）**: 2列グリッド、角丸16px。6種（未架電/不通/担当者不在/ヒアリング済/アポ獲得/お断り）。各ステータス色の淡い背景+濃い文字。アイコンは線アイコン or 塗り円の2系統（`STATUS_ICON` 定義）。選択時は `currentColor` の2.5pxリング。
- **保存**: ステータス未選択だとボタンは `.disabled`（グレー）。保存すると `.ok`（緑）＋「保存しました」、850ms後に詳細へ。保存時に履歴へ追記し、リアルタイム配信（broadcast）する。

### 5. 履歴（HistoryScreen）
- **目的**: チーム全体の架電アクティビティを時系列フィードで見る。
- **レイアウト**: 全医院の履歴を `at` 降順にフラット化したフィード。各行=担当者アバター（円・イニシャル）+ 医院名 + 時刻 + Badge + 担当者名 + メモ。行区切り1px。

### 6. ダッシュボード（DashScreen）
- **目的**: 週次の集計を見る。
- **レイアウト**: ①サマリーカード（`--card`背景・角丸16px：架電数/ヒアリング/アポ獲得の3分割、罫線、ヒアリング率/アポ率の2分割）②ステータス別サマリー（色ドット+ラベル+件数+chevron、タップで一覧へ）③架電数の内訳（水平バーチャート、ラベル72px幅+トラック+数値、バーはステータス色）。

## Interactions & Behavior
- **ナビゲーション**: 画面状態は `view = { screen, clinicId }`。`nav(screen, clinicId)` で遷移。`screen === 'next'` は「次に架電すべき医院」を算出して詳細へ。遷移時にスクロール位置をトップへ。
- **「次に架電すべき医院」算出（nextToCall）**: `status === 'none'`（未架電）を最優先 → 無ければ `none/noanswer/absent`（フォローアップ対象）→ それも無ければ先頭。
- **電話発信**: 電話関連は実 `tel:` リンク（番号からハイフン除去）。
- **音声入力**: モックではボタンでメモ欄に定型文を追記するだけ（実装時は端末の音声入力APIへ）。
- **リアルタイム共有（重要な体験の核）**:
  - 同タブ内で「同僚ボット」が4.5〜13秒間隔でランダムに他医院のステータスを更新 → トースト通知（左上、`toastIn` アニメ）＋該当カードがフラッシュ。
  - 「プレゼンス」（誰が何をしているか）も3.5〜6秒間隔で更新され、ホーム上部のピルに反映。
  - **複数タブ間同期**: `BroadcastChannel('dentia-sync-v1')` で更新・プレゼンスを他タブへ配信。別タブで開くと相互に反映される。
  - 実装時は、このボット/BroadcastChannelを**実際のリアルタイムバックエンド（WebSocket / Firebase / Supabase Realtime 等）に置き換える**こと。ボットはあくまでデモ用の擬似同僚です。
- **アニメーション**: プログレスバー/水平バーは `width` を `0.5s ease`。カードフラッシュ `flashPop 0.5s`。トースト `toastIn 0.35s cubic-bezier(.2,.9,.3,1.3)`。プレゼンスドット/トーストドットは `pulse`。ボタン押下は `transform: scale(.985〜.97)`。

## State Management
ルートの `App` コンポーネントが集中管理（`app/app.jsx`）：
- `clinics`: 医院配列（`window.Dentia.CLINICS` を初期値にディープコピー）。各医院に `history[]` を保持。
- `presence`: 同僚の在席/操作リスト（自分=meは除外）。
- `toast`: 現在表示中のトースト（3.6sで自動消去）。
- `flashId`: フラッシュ中の医院ID（1.4sで解除）。
- `view`: `{ screen, clinicId }`。
- `t`（tweaks）: アクセント色・フォント・情報密度・ダーク・ボットON/OFF（`useTweaks`／プロトタイプ用のデザイン調整パネル。本番では不要なら削除可）。
- **更新の流れ**: `applyUpdate(u)` がローカル state に反映し `applyUpdateToClinics()` で該当医院の status/owner/lastMemo/updatedAt と history 先頭を更新、同時に realtime へ broadcast。リモート更新 `onRemote` は state 反映＋（自分以外なら）トースト＆フラッシュ。

## Data Model
医院オブジェクト（`app/data.js` の `CLINICS`）:
```
{
  id, name, tel, prefecture, city, address, hours,
  status,            // 'none'|'noanswer'|'absent'|'heard'|'appo'|'reject'
  owner,             // userId | null
  lastMemo,
  updatedAt,         // ISO | null（未架電）
  nextSchedule,      // 表示用文字列 | null
  history: [ { at(ISO), who(userId), status, note } , ... ]  // 新しい順
}
```
ステータス定義（`STATUS` / 表示順 `STATUS_ORDER`）と担当者（`USERS`：id/name/short/color）も `data.js` 参照。モックの「現在時刻」は `NOW = 2026-06-05T09:41`、経過表示は `fmtAgo()` がこれ基準で算出。本番では実時刻に置き換える。

## Design Tokens
配色は `Dentia.html` の `:root` に定義（ライト）。ダークは `.dark` で上書き。

**ブランド / ベース**
- アクセント（既定）= **水色 `#6ab3e0`**。トークン名は歴史的経緯で `--teal` のまま（値が水色に差し替わる）。他候補 `#0c8c8b / #8aa9e6 / #b596de / #ec9bc1`。※Tweaksで変更可
- テキスト `--ink #1f2a33` / `--ink2 #3a4750` / `--muted #6b7682` / `--muted2 #9aa4ae`
- 罫線 `--line #eceff1` / `--line2 #e3e7ea`
- 面 `--page #ffffff` / `--card #f6f8f9` / `--field #ffffff`

**ステータス色（背景bg / 文字fg のペア）**
- gray（未架電） `#9aa4ae` / bg `#eef1f3` / fg `#69737d`
- amber（不通） `#eab308` / bg `#fdf2d4` / fg `#c08a05`
- orange（担当者不在） `#f3722c` / bg `#fde6d6` / fg `#df6320`
- cyan（ヒアリング済） `#0c8c8b` / bg `#d3edec` / fg `#0a8584`
- green（アポ獲得） `#22b14c` / bg `#d9f3df` / fg `#1a9a45`
- red（お断り） `#ef5350` / bg `#fce0df` / fg `#dc4843`

**ダークモード（.dark 上書き、抜粋）**
- `--page #161c22` / `--card #1d242b` / `--ink #eef2f4` / `--line #262f37` 他（CSS参照）

**タイポgrafi**
- フォント: `Noto Sans JP`（既定）。他に Zen Kaku Gothic New / Zen Maru Gothic / M PLUS Rounded 1c を選択可（Tweaks）。本番は端末/プロダクトの日本語フォント方針に合わせる。
- 主なサイズ: 進捗大数字34px/900、医院名（ホーム）20px/800・（詳細）22px/800・（一覧カード）18px/800、本文14px、補助12〜13px。

**角丸 / 影 / 余白**
- 角丸: ボタン14px、カード14〜16px、チップ/ピル999px、入力12〜14px。
- 影: プライマリボタン `0 6px 16px (teal 28%)`、FAB `0 8px 18px (teal 34〜38%)`、トースト `0 12px 30px rgba(20,28,36,.18)`。
- 情報密度（Tweaks）: compact / regular / comfy で `.sec` paddingやカードgap等が変化（`.dens-*` 参照）。本番は regular を基準に。
- デバイス枠: 390×844、角丸46px、11px枠（プロトタイプの飾り。本番では不要）。
