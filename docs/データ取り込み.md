# 歯科医院データの取り込み（厚労省オープンデータ）

医院マスタは、厚生労働省「医療情報ネット」の**歯科診療所オープンデータ**から取り込みます。
APIではなく**ZIP/CSVのダウンロード**で完結します（キー・課金・規約上の保存制限なし）。

- 一覧: [医療情報ネットのオープンデータ｜厚生労働省](https://www.mhlw.go.jp/stf/seisakunitsuite/bunya/kenkou_iryou/iryou/newpage_43373.html)

## ⚠️ 重要な制約：電話番号は含まれない

厚労省データには **名称・所在地・診療時間・休診日**は含まれますが、**電話番号は含まれません**。
架電CRMの中心は電話なので、`phone` は取り込み時 NULL のままになります。番号は別途補完してください：

- 医院詳細画面から**手入力**（架電準備のついでに）
- 将来的に **Google Places API**（名称＋住所で検索 → 電話番号）で補完するスクリプトを追加
- タウンページ等の**ライセンスデータ**

## データの形

歯科は2ファイルに分かれており、`ID` で結合します。

| ファイル | 主な項目 |
| --- | --- |
| `03-1_dental_facility_info_*.csv`（施設票） | ID・正式名称・都道府県/市区町村コード・所在地・休診日 |
| `03-2_dental_speciality_hours_*.csv`（診療時間票） | ID・診療科目・診療時間帯・曜日別の診療開始/終了時刻 |

診療時間票は1医院につき「診療科目 × 診療時間帯」で複数行あり、本スクリプトが
曜日別の `hours`(jsonb) に変換・重複除去します（昼休みは区間分割で表現）。

## 手順

### 1. ダウンロード＆解凍

```bash
mkdir -p data && cd data
BASE=https://www.mhlw.go.jp/content/11121000
curl -LO $BASE/03-1_dental_facility_info_20251201.zip
curl -LO $BASE/03-2_dental_speciality_hours_20251201.zip
unzip -o '*.zip'
cd ..
```

（最新の時点別URLは上記の厚労省ページで確認。`data/` は .gitignore 済み）

### 2. まず dry-run で確認

```bash
pnpm import:clinics \
  --facility data/03-1_dental_facility_info_20251201.csv \
  --hours    data/03-2_dental_speciality_hours_20251201.csv \
  --prefecture 東京都 --city 新宿区 --limit 5 --dry-run
```

### 3. Supabase へ投入

サービスロールキー（RLSをバイパスする管理キー。Supabase ダッシュボード → Project Settings → API）が必要です。

```bash
NEXT_PUBLIC_SUPABASE_URL=https://YOUR.supabase.co \
SUPABASE_SERVICE_ROLE_KEY=YOUR-SERVICE-ROLE-KEY \
pnpm import:clinics \
  --facility data/03-1_dental_facility_info_20251201.csv \
  --hours    data/03-2_dental_speciality_hours_20251201.csv \
  --prefecture 東京都
```

## オプション

| フラグ | 説明 |
| --- | --- |
| `--facility <csv>` | 施設票CSV（必須） |
| `--hours <csv>` | 診療時間票CSV（必須） |
| `--prefecture <名>` | 都道府県で絞り込み（例 `東京都`）。全国は約5.3万件あるため通常は指定推奨 |
| `--city <名>` | 市区町村で絞り込み（例 `新宿区`） |
| `--limit <N>` | 先頭N件のみ |
| `--dry-run` | DB書き込みせずサンプルと件数のみ表示 |
| `--source <名>` | `source` 列の値（既定 `mhlw`） |

## 冪等性（再取り込み）

`external_id`（厚労省の `ID`）を衝突キーに **upsert** します。新しい時点のデータで
再実行すると、名称・住所・診療時間は更新され、**架電の進捗（status・latest_memo・
assigned_to・next_action_at）と手入力した phone は保持**されます。
（取り込みスクリプトはこれらの列をペイロードに含めないため）

> `external_id` 列は `supabase/schema.sql` に追加済み。スキーマ未適用の場合は再実行してください。
