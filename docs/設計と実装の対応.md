| 役割 | ファイル |

## 設計からの差分・拡張メモ
- `middleware.ts` → `proxy.ts`（Next.js 16 対応）。
- profiles の RLS に「自分の行のみ insert/update 可」を追加（初回ログイン時の upsert のため）。設計書の「認証済みは全件可」の方針内で、書き込みのみ自分の行に限定。
- clinics に取り込み・補完用の列を追加: `external_id`(冪等キー), `lat`/`lng`(座標照合), `place_id`, `phone_source`, `phone_verified`。
- `phone_candidates` テーブル（電話番号の確認キュー）を追加。
- **背景**: 厚労省データに電話番号が無いため、設計書付録の「Google Places 連携」を、
  誤マッチ防止（座標照合）と人の確認（キュー＋未確認フラグ）を組み込む形で具体化した。
